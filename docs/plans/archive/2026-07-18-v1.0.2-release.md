## Known issues encountered

### 1. `ascApiKeyPath` missing from `eas.json`
`eas.json` `submit.production.ios` was missing `ascApiKeyPath`. Added:
```json
"ascApiKeyPath": "credentials/AuthKey_XT32ZS8GHR.p8"
```

### 2. `eas build --auto-submit` interactive prompt (EAS CLI 21.0.0)
`--non-interactive` did not suppress the Apple Team ID prompt during the
submit phase. Workaround: Ctrl+C after build enqueued, run `eas submit`
separately.

### 3. `eas-app-store-kit` age-rating 409 on re-submission
The kit 409-errors on `PATCH /ageRatingDeclarations/{id}` for apps that
are already live, because the `AppInfo` is immutable. The age rating carries
over automatically. Submitted directly via the ASC API instead.
See `docs/release-to-app-store.md §8` for the full workaround.

### 4. `whatsNew` required before review submission
Adding the version to a review submission fails with
`ENTITY_ERROR.ATTRIBUTE.REQUIRED` until `whatsNew` is set on the
`appStoreVersionLocalization`. Set before calling `POST /reviewSubmissionItems`.
